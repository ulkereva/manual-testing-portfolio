| **Severity** | Medium |
| **Priority** | Medium |
| **Status** | Open |

---

## Summary
When a locked-out user attempts to log in, the error message shown is a long technical string that does not clearly explain *why* login failed, causing confusion for end users.

---

## Steps to Reproduce

1. Open https://www.saucedemo.com
2. Enter email: `lockedoutemail@gmail.com`
3. Enter password: `secret_sauce`
4. Click the **Login** button

---

## Expected Result
A clear, user-friendly message such as:  
> *"Your account has been locked. Please contact support."*

---

## Actual Result
The error message displayed is:  
> *"Epic sadface :( Sorry, this user has been locked out."*

While this technically conveys the account is locked, the informal tone ("Epic sadface") and lack of next-step guidance (no support link, no contact info) is unprofessional in a real production environment.

---

## Impact
- Users do not know who to contact or what to do next
- Poor UX for non-technical users
- In a real app, this could increase support ticket volume

---

## Notes
- This is a known intentional behavior on saucedemo.com (it is a test site)
- In a real application, this would be a valid bug to report and fix
- Severity is Medium (not Critical) because login itself is blocked correctly — only the message quality is the issue
