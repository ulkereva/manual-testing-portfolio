# BUG-003 — Checkout proceeds with empty First Name field

| Field | Detail |
|-------|--------|
| **Bug ID** | BUG-003 |
| **Related Test** | TC-013 |
| **Module** | Checkout |
| **Reported By** | Ulker Aliyeva |
| **Date** | 2024-06-18 |
| **Environment** | Chrome 124, Windows 11, https://www.saucedemo.com |
| **Severity** | Major |
| **Priority** | High |
| **Status** | Open |

---

## Summary
The checkout form allows the user to proceed to the Order Overview page even when the **First Name** field is left completely empty. No validation error is shown.

---

## Steps to Reproduce

1. Log in as `standardemail@gmail.com` / `secret_sauce`
2. Add any product to the cart
3. Click the cart icon → click **Checkout**
4. On the "Your Information" page:
   - Leave **First Name** field **empty**
   - Enter Last Name: `Surname`
   - Enter Postal Code: `AZ1234`
5. Click **Continue**

---

## Expected Result
Validation error should appear:  
> *"First Name is required"*  
User should NOT be able to proceed to the Overview page.

---

## Actual Result
The form submits successfully and the user is taken to the **Checkout: Overview** page with no error shown. The order can even be completed without a first name.

---

## Impact
- Order data is incomplete — no customer first name recorded
- Could cause issues in order fulfillment / shipping
- Allows users to submit orders anonymously without proper identification
- In a real e-commerce system, this would be a **critical** data integrity bug

---

## Notes
- Severity is **Major** (not Critical) because checkout still functions — but data is invalid
- Priority is **High** because incomplete order data directly impacts business operations
- The Last Name and Postal Code fields DO have validation — only First Name is missing it
- This appears to be an intentional defect planted on saucedemo.com for testing practice
